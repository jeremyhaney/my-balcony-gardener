PROJECT INSTRUCTIONS: My Balcony Gardener (MBG)

Architecture Plan: Split-Stack IoT Web Application (LOCKED)

These instructions define the exact technical structure. Do not deviate without explicit revision to this document.

1. SYSTEM OVERVIEW

MBG is a split-stack system designed to monitor and control balcony plant watering using IoT best practices.Each system component is isolated and communicates through Supabase via HTTPS using REST APIs.

2. SYSTEM COMPONENTS

A. ESP32 (Device Layer)

Uses the DOIT ESP32 DEVKIT V1

Programmed via PlatformIO using the Arduino framework

Logs temperature, humidity, soil moisture, and watering data every 5 seconds

Sends JSON data directly to Supabase REST API using HTTPS and bearer token authentication

No local dashboard is hosted from the ESP32

B. Supabase (Backend Layer)

Serves as the central backend for all data

Active row-level security (RLS) is in place

Primary table: sensor_logs

Fields: timestamp, temperature, humidity, moisture, watering, etc.

ESP32 is authorized via service role token to post data

Frontend is restricted to read-only access through public API key scoped by RLS policy

C. React Frontend (UI Layer)

Built with Vite + React

Uses Supabase JavaScript client to pull sensor log data from sensor_logs

Displays:

Current sensor values

Real-time graphing via HTML5 Canvas or charting library

System status (online, offline, last watering, etc.)

No mock data is used in production

D. Cloudflare Pages (Frontend Hosting Platform)

Final site is deployed to Cloudflare Pages

Public-facing site is accessed via Cloudflare Tunnel

After each production build (npm run build), contents of /dist/ are deployed to the Pages platform

Tunnel URL is used for development and testing until domain is mapped

3. DEVELOPMENT AND DEPLOYMENT WORKFLOW

ESP32

Code location: esp32_firmware/src/main.cpp

Upload via PlatformIO (COM3 on BigJer2)

Posts to Supabase endpoint:

POST https://<project>.supabase.co/rest/v1/sensor_logs

Supabase

Manages:

sensor_logs table

RLS policies

Authentication tokens

RLS is active and must remain enforced

Supabase keys:

Service role key used only by ESP32 (never exposed in frontend)

Public anon key used only in frontend for read-only access

React Frontend

Source code: mbg_dashboard/src/

Components:

LiveStats.tsx – Fetches data and renders charts

App.tsx – Main UI structure

Env file .env holds Supabase project URL and anon key

Deployment

From root of frontend:

npm run build

Deploy contents of /dist/ to Cloudflare Pages (configured project)

Access site via Cloudflared tunnel:

https://<unique-subdomain>.trycloudflare.com
